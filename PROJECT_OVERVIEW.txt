🏗️ ARCHITECTURE OVERVIEW
=========================

Frontend (React)
├─ Home Page ──────> Landing with features
├─ Login Page ──────> Register or login
└─ Dashboard ──────> Pipeline cards
    ├─ New Column (Yellow)
    ├─ Contacted Column (Blue)
    └─ Converted Column (Green)
        └─ LeadCard (individual)
            └─ Notes Modal (timeline)

Backend (Node + MongoDB)
├─ POST /api/auth/register ──> Create user
├─ POST /api/auth/login ────> Generate JWT token
├─ GET /api/leads ─────────> Fetch all leads
├─ POST /api/leads ────────> Create lead
├─ PUT /api/leads/:id ─────> Update lead status
├─ DELETE /api/leads/:id ──> Delete lead
└─ POST /api/leads/:id/notes > Add note

MongoDB Collections
├─ users (auth + roles)
└─ leads (clients + pipeline status + notes)

📱 USER FLOWS
=============

1. NEW USER (Registration)
   Home → Click "Login/Register" → Register Tab → Sign up → Auto-login → Dashboard

2. EXISTING USER (Login)
   Home → Click "Login/Register" → Login Tab → Enter credentials → Dashboard

3. ADD LEAD
   Dashboard → "+ New Lead" Button → Fill form → Create Lead → Appears in "New" column

4. MANAGE LEAD
   Dashboard → Drag card between columns → Status updates → Add notes → Track aging

5. VIEW NOTES
   Dashboard → Click "Notes (n)" on card → See timeline → Add new note → Save

📊 SAMPLE DATA FIELDS
====================

User Schema:
├─ _id (ObjectId)
├─ name (string)
├─ email (string, unique)
├─ password (hashed)
├─ role (admin/staff)
└─ createdAt (timestamp)

Lead Schema:
├─ _id (ObjectId)
├─ clientName (string)
├─ email (string)
├─ phone (string)
├─ roomType (enum: Kitchen/Living/Bedroom/Bathroom/Office)
├─ estimatedBudget (number, ₹)
├─ status (New/Contacted/Converted)
├─ notes (array of {text, createdAt})
├─ lastNoteAddedAt (timestamp)
├─ userId (reference to User)
└─ createdAt (timestamp)
